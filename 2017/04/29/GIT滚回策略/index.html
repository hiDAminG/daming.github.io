GIT滚回策略

  • GIT的日常

为什么要剿BUG！你们想想，你们带着老婆，坐着火车，吃着火锅，唱着唱歌。突然就让BUG给劫了！
所以BUG是必须要剿的！！

大明最近遇到了这样的烦心事，让吃着火锅唱着歌就让BUG给截了。但是大明也不怕遇到事，遇到BUG改了便是。但是福无双至祸不单行，现在开发的版本有已经与之前的版本相去甚远。
没关系git就是为这个而生的~可以轻松的让你滚回，回温当初的美好~~

想必大家对GIT还是不陌生的，但是孔乙己过来问你回滚有几种滚法你可能就有点说不清了。

其实回滚也只有一种滚法

言归正传，我们从最简单的情况说起。
git reset 滚回~~
git reset 中有很多参数 其中有一个--hard 参数；

使用

便可以回到当前HARD 版本的上一个版本，HRAD^^可以回滚上上个版本了；
以此类推，滚回到前到少个就加几个 ^ 也可以用HARD~100滚回到前100个版本；

利用

命令我们可以查到版本列表，找到每个版本的 'commit id'，如“ede24c70fd3baca05a76cfd256d8804d5df8b864”;

这时只要将参数换成'commit id' 就可滚回到相应版本：

这里参数可以不用写全id，git 会自己匹配将对应的，但是只写1、2位可就后果自负了！！

到这里你是不是觉得git滚回很好很强大，但是这里不是他的全部；

使用用git reset 是整个版本整个版本的回滚，那我只想要回滚一个文件怎么办？？
可以使用

将某个文件滚回某个版本，但是无并不建议这样使用！reset命令滚回后时间线之后的文件删除了！！
所以一旦文件被提交到了远端，就慎用reset滚回，以防出现不想见的情况。

那应该怎样安全的滚回呢？？git 也为你准备好了切糕'CHECKOUT'。
git checkout命令就可以为你办到！！
git checkout不但可以在不同 分支 间穿梭，还可以让 文件 在不同版本建穿梭，是不是深深爱上了这个穿梭小能手~~

让文件在不同版本中穿梭很简单，和之前相似：

就可以轻松的穿梭在版本的长河里面了。

但是凡事就怕个但是。
在实际操作过程中，我们往往会开辟不同分支，在不同分支之间游走如果不进行commit，就会报错。强行切换 工作区 未保存修改就会丢失。
实际情况是你遇到一个紧急BUG要修改，手头的工作做到了一半，刀架脖子上了，哪还有工夫commit！！

没事git也早就为我们准备好了工具：
